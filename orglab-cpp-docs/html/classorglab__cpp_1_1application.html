Detailed Description

Wrapper for IOApplication instance

Singleton class to ensure only once instance is ever created

Because OrgLab is essentially single-threaded (only one Origin project may be open at a time), when calling methods for an instance of this class in a multithreaded application, a mutex should be used to ensure singleton is only accessed by one thread at a time
